Result:

- the old audit still mattered, because the claimed public-export hardening is not actually present in the current repo state

### 2. Public docs still expose internal Doppler details

`docs/RELEASE.md` still contains internal Doppler project/config/key naming.

Examples found during the refresh audit:

- `kombination`
- `prd`
- `GITHUB_PAT`

### 3. CI still uses outdated Go version

The current stackKits workflows still pin Go `1.24` in CI/release, while the platform standard requires `1.25+`.

Impacted workflows:

- `.github/workflows/ci.yml`
- `.github/workflows/release.yml`

### 4. Release documentation is internally inconsistent

`docs/RELEASE.md` describes synced artifacts and workflow behavior that do not match the actual repository layout.

This means the doc is currently not trustworthy as an operational source.
