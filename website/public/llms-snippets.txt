# StackKits Agent Snippets

## Autonomous BaseKit

```bash
stackkit init base-kit --non-interactive --admin-email admin@example.com
stackkit prepare --dry-run
stackkit validate
stackkit generate --force
stackkit plan
stackkit apply
stackkit verify --http --json
```

## Inspect

```bash
stackkit status --json
stackkit verify --http --json
stackkit logs list --json
stackkit doctor --json
```

## MCP Config

```bash
stackkit agent mcp-config --client codex --mode docs,local,server
```

Website discovery:

```text
GET https://stackkit.cc/openmcp.json
GET https://stackkit.cc/getting-started/installation-processes.md
```

Native local connector:

```text
[mcp_servers.stackkit]
command = "stackkit-mcp"
args = ["--mode", "docs,local,server"]
```

Local endpoint exposed by stackkit-server after install:

```text
POST http://localhost:8082/mcp
GET  http://localhost:8082/openmcp.json
```

Protected remote MCP target:

```text
POST https://<protected-target>/mcp
GET  https://<protected-target>/openmcp.json
```

Agents should configure one MCP connection named `stackkit`. Remote MCP is the same connector for installed StackKit day-2 operations through Streamable HTTP. WebSocket would be a custom gateway, not the default StackKits surface.
