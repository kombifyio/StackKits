# kombify-StackKits - Roadmap

## Current State (v1.0.0-beta - 2026-05-09)

StackKits is in the V5 production-readiness track with V6 planning active. The current reliable baseline is the Base Kit fresh-Ubuntu verification path: `prepare -> init -> generate -> apply`, with OpenTofu installed on the target by `stackkit prepare` and Docker Hub authentication required before treating the full container smoke as a release gate. Base Kit is not test-user-ready for Monthly Runtime until Standard and Premium both prove `admin-bootstrap`, `login-gateway`, `security-baseline`, runtime status, SSH enable/disable, stop/start, and cleanup/decommission.

The immediate focus is release-readiness and standards alignment: CUE remains the technical contract source of truth, Beads is the source of truth for active roadmap items, and `ROADMAP.md` is a generated read-view for near- and medium-term work.

Release gating starts with **StackKits-j62**: leaked dev credential rotation and full secret-history remediation remain the P0 blocker before any public release handoff.

## Long-Term (backlog)

- **CUE Decision Logic as the deployment engine:** Complete the CUE binding path so module contracts, context defaults, compatibility gates, security constraints, and generated OpenTofu shape all flow from CUE-backed resolution instead of parallel Go logic.
- **Production-grade default modules:** Promote `security-baseline`, `admin-bootstrap`, and `login-gateway` only after the fragmented generation path can validate their host, identity, route, secret, backup, and smoke-test contracts.
- **Modern Homelab and HA Kit:** Keep these behind explicit roadmap work until local/cloud placement, VPN or split-DNS behavior, quorum/failover, restore, and update sequencing have enforceable contracts.
- **TechStack orchestration:** Treat StackKits as the releaseable CLI/API contract and let kombify TechStack consume the same CUE, registry, and OpenAPI surfaces rather than introducing separate defaults.
- **Public OSS release hygiene:** Continue the private-source to curated-public mirror model with allowlisted exports, leak validation, public release builds, and post-publish smoke checks.

## Won't Do

- **No hand-authored generated rollout artifacts:** OpenTofu, Compose, tfvars, scripts, and release snapshots remain generated outputs. If output needs manual edits, the CUE contract, generator, or registry mirror is incomplete.
- **No database override of live CUE contracts:** The kombify database is the catalog, registry, operations, wizard, tenant, and lifecycle mirror; it does not replace CUE as the technical source of truth.
- **No public release from the private source repo:** Public tags, releases, and downloadable assets are produced through the curated `kombifyio/stackKits` release surface.
- **No Kubernetes path in Base Kit, Modern Homelab, or HA Kit v1:** Docker-first remains binding for the current StackKit line. Kubernetes can only return as a separate future StackKit after an accepted ADR.
